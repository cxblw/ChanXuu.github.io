jenkins和nginx实现前端自动构建

ChanXuu 2020-09-10 jenkins nginx

# 目标

本地push代码到GitHub/Gitee 上时,Webhook自动触发,将GitHub/Gitee上的代码通过jenkins的构建,实现 打包代码-部署到服务器 这一过程

# 准备工作

  1. GitHub/Gitee 的账号
  2. 服务器一台
  3. 服务器安装JAVA环境
  4. 服务器安装Nginx并启动
  5. 服务器安装Jenkins并启动

# 步骤

# 1. 登录服务器更新系统软件(可有可无)

 yum update
1

# 2. 安装Java和git

 yum install java
 yum install git
1
2

# 3. 安装nginx并启动

 yum install nginx //安装
 service nginx start //启动
1
2

出现 Redirecting to /bin/systemctl start nginx.service 表示开启成功, 访问http://你的ip/,成功会出现nginx默认页面

# 4. 安装Jenkins

 wget -O /etc/yum.repos.d/jenkins.repo http://pkg.jenkins-ci.org/redhat/jenkins.repo

rpm --import https://jenkins-ci.org/redhat/jenkins-ci.org.key 

 yum install jenkins //完成之后直接使用 yum 命令安装 Jenkins

 service jenkins restart  //启动 jenkins
1
2

  • jenkins启动成功后默认的是8080端口，输入你的服务器 ip 地址加8080 端口就可以访问了。
  • 在终端输入 cat /var/lib/jenkins/secrets/initialAdminPassword 可以获得jenkins的管理员密码
  • 推荐通用插件安装,接下来等待它安装即可

如果 yum install jenkins 后出现 warning: rpmts_HdrFromFdno: Header V4 DSA/SHA1 Signature, key ID d50582e6: NOKEY 类似的错误,可以尝试 yum install --nogpgcheck jenkins 掉gpg检查

# 6. 安装nodejs插件,并配置

# 7. 创建任务

到了这一步,我们已经完成了Jenkins将代码push并打包的过程

# 8. 安装Publish Over SSH 插件，实现服务器部署

  1. 先安装插件,安装完成后,进入 Manage Jenkins -> Configure System -> Publish over SSH

填写完后点击test，出现Success说明配置成功

  1. 在刚才的testJenkins工程中配置 构建后操作 ，选择 send build artificial over SSH

还要在构建中添加压缩dist目录命令

填完后执行构建。成功后可以发现目标服务器发现项目目录下有了要运行的文件,直接访问域名就发现可以访问了

接下来实现开发本地push代码到github上后，触发Webhook，jenkins自动执行构建。

# 9. 触发Webhook

在刚才的testJenkins工程中点击构建触发器中选择 Generic Webhook Trigger ，填写token

  • github配置Webhook 选择github项目中的 Settings->Webhooks>add webhook

  • gitee配置Webhook

最后修改项目内容,使用git提交,jenkins就会自动帮我们push代码,打包,然后部署到服务器了,等jenkins自动构建完成后,再访问项目域名,会发现内容已经改变了

参考文章 https://segmentfault.com/a/1190000019212628 感谢作者的文章,让我跟着步骤学会了前端项目的自动部署
